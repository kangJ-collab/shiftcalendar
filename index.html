구글 스프레드시트 + GAS 설정
이 설정은 최초 1회만 하면 됩니다. 아이폰에서도 가능하지만 Apps Script 메뉴가 잘 안 보일 수 있으므로 처음 설정은 PC 또는 Safari의 데스크탑 웹사이트 모드를 권장합니다.
1. 구글 시트 만들기
설정창의 ① 구글 시트 홈 열기를 누릅니다.
구글 시트 홈에서 빈 스프레드시트 또는 +를 눌러 새 시트를 만듭니다.
문서 이름은 예: 교대달력 백업으로 지정합니다.
2. Apps Script 열기
방금 만든 스프레드시트 안에서 상단 메뉴의
확장 프로그램 → Apps Script를 누릅니다.

아이폰에서 메뉴가 안 보이면 Safari 주소창의 가가 → 데스크탑 웹 사이트 요청을 사용하거나 PC에서 진행하세요.
3. GAS 코드 붙여넣기
교대달력 설정창의 ② GAS 코드 복사를 누릅니다.
Apps Script 편집기에 기본으로 들어 있는 코드를 모두 지우고 붙여넣은 뒤 저장합니다.
4. 데이터베이스 양식 자동 생성 및 권한 승인
Apps Script 상단의 함수 선택란에서 setupDatabase를 선택한 뒤 실행합니다.

처음 실행하면 아래처럼 영어 화면으로 Google hasn't verified this app가 표시될 수 있습니다. 본인이 직접 만든 스크립트이므로 당황하지 말고 다음 순서대로 진행합니다.

① 왼쪽 아래 Advanced(고급)를 누릅니다.
② 아래에 나타나는 Go to 제목 없는 프로젝트 (unsafe)를 누릅니다.
프로젝트 이름은 사용자가 정한 이름에 따라 다르게 표시될 수 있습니다.
③ 다음 권한 화면에서 Continue(계속)를 누릅니다.


승인 후 다음 시트가 자동 생성됩니다.
config / appData / settings / attendance / teamHistory / salaryOverrides / companyHolidays

같은 화면이 다시 나오지 않도록 하려면 이후에는 같은 구글 계정으로 작업하세요.
5. 웹 앱으로 배포
Apps Script 오른쪽 위에서
배포 → 새 배포 → 유형 선택 → 웹 앱 순서로 들어갑니다.

실행 사용자: 나
액세스 권한: 모든 사용자

배포를 누르면 여러 항목이 표시될 수 있습니다. 여기서 배포 ID를 복사하면 안 됩니다.

반드시 웹 앱 URL 항목의 주소를 복사해야 하며, 주소 끝은 보통 /exec로 끝납니다.

예시:
https://script.google.com/macros/s/......../exec
6. 교대달력에 URL 입력
교대달력 설정창의 GAS 웹 앱 배포 URL에 /exec로 끝나는 웹 앱 URL을 붙여넣습니다.
연결정보 저장 → 연결 테스트 순서로 누릅니다.
7. 백업과 복원
연결이 성공하면 지금 백업으로 현재 데이터를 구글 시트에 저장할 수 있습니다.
다른 기기에서는 같은 GAS URL을 입력한 뒤 백업 불러오기를 사용합니다.
